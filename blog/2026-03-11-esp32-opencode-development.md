## 硬件接线

根据生成的代码，接线非常清晰：

```
ESP32 引脚分配：
GPIO 4  → DS18B20 数据引脚
GPIO 5  → DHT11 数据引脚
GPIO 16 → 继电器控制引脚
GPIO 21 → OLED SDA
GPIO 22 → OLED SCL
```

**关键注意事项：**
- ✅ DS18B20 需要 4.7kΩ 上拉电阻
- ✅ 共地（所有 GND 连在一起）
- ✅ 继电器接 VIN（5V），传感器接 3V3

## 遇到的坑

### 坑 1：DS18B20 总是返回 -127°C

**现象：** 水温传感器读取失败

**解决对话：**
```
我：DS18B20 总是返回 -127°C，这是我的接线代码，请帮我看看

opencode：看到问题了。DS18B20 需要 4.7kΩ 上拉电阻连接到 3V3，
否则无法正常通信。另外检查：
1. 确认 DAT 引脚连接到 GPIO 4
2. 确认 VCC 接 3V3（不是 5V）
3. 确认共地
```

**结果：** 加上拉电阻后立即正常。

---

## 对话技巧总结

### ✅ 正确做法

**1. 逐步添加功能**
```
❌ 一次性描述所有需求
✅ 先生成基础框架，再逐步添加功能
```

**2. 描述问题时提供上下文**
```
❌ 代码报错了，帮我看看
✅ 编译时报错 "undefined reference to 'setup()'"，这是 main.cpp
```

**3. 明确告知技术栈**
```
✅ 使用 PlatformIO + ESP32 + Arduino 框架
✅ 使用 AsyncWebServer 库（更好的并发性能）
```

### ❌ 避免踩坑

**1. 不要一次性描述所有需求**
   - AI 可能遗漏细节
   - 代码生成质量下降
   - 后续修改成本高

**2. 不要忽略错误信息**
   - 具体的错误信息能帮助 AI 快速定位
   - 提供代码片段让 AI 上下文更清晰
