# 快速问答

OpenCode 的快速问答功能让您能够通过简单的对话快速获得准确的答案和解决方案。本文将详细介绍问答的最佳实践、提问技巧、上下文管理、多轮对话以及常见问答场景。

通过掌握快速问答技巧，您可以大幅减少信息搜索时间，快速解决工作中遇到的各种问题。

## 快速问答概述

快速问答是 OpenCode 的核心功能之一，它让用户能够：

- **自然语言提问**：无需了解技术细节，用日常语言描述问题
- **实时获得答案**：基于强大的 LLM 能力，快速生成准确回答
- **上下文理解**：记住对话历史，理解问题上下文
- **多源信息整合**：结合代码库、文档、网络等多方面信息

## 提问技巧

### 1. 明确问题

好的提问应该：

- **具体明确**：避免模糊的描述
- **提供背景**：说明问题出现的场景
- **明确目标**：说明你想要什么样的答案

**示例：**

❌ **差的提问：**
```
我的代码不工作
```

✅ **好的提问：**
```
我的 React 组件在用户点击按钮后没有更新状态，即使我已经调用了 setState。这是我的组件代码，请帮我看看问题在哪里？
```

### 2. 提供上下文

使用 `@` 符号引用代码库中的文件：

```
在 @packages/auth/src/login.ts 中，身份验证失败时如何处理错误？
```

使用 `#` 符号引用代码片段：

```
在 @utils/request.ts 中的 fetch 函数里，超时设置是多少？
```

### 3. 分步提问

对于复杂问题，可以分步提问：

```
第 1 步：请解释一下 React 的 useEffect 钩子是什么？

第 2 步：我的组件中使用 useEffect 时遇到了问题，请帮我看看。

第 3 步：如何修复这个问题？
```

## 上下文管理

### 对话历史

OpenCode 会记住对话历史，理解上下文关系：

```
你：什么是 React 的虚拟 DOM？

OpenCode：[详细解释虚拟 DOM]

你：它有什么优势？

OpenCode：[基于之前的解释，回答虚拟 DOM 的优势]
```

### 上下文限制

当对话过长时，OpenCode 会自动压缩旧对话，但保留关键信息。

**提示**：如果发现 OpenCode 忘记了之前的内容，可以重新提供关键信息。

## 常见问答场景

**目标**：理解复杂的代码逻辑

**提问方式：**

**提问方式：**

```
这是相关代码，请帮我看看问题在哪里？
```

**OpenCode 的回答会包括：**
- 错误原因分析
- 可能的解决方案
- 代码修改建议
- 预防措施

**目标**：了解行业标准做法

**提问方式：**

**OpenCode 的回答会包括：**
- 常见的验证模式
- 推荐的库和工具
- 性能优化建议
- 代码示例

### 场景 4：技术决策

**目标**：在多个方案中做出选择

**提问方式：**

**OpenCode 的回答会包括：**
- 各方案的对比表格

## 多轮对话技巧

### 1. 逐步深入

从简单问题开始，逐步深入细节：

```
第 1 轮：什么是 React 的 Context API？

第 2 轮：如何创建一个 Context？

第 3 轮：如何在组件中使用 Context？

第 4 轮：Context 的性能如何优化？
```

### 2. 追问细节

对不清楚的地方追问：

```
你：请解释一下 TypeScript 中的泛型

OpenCode：[解释泛型的概念和语法]

你：能给我举一个实际的使用例子吗？

OpenCode：[提供实际代码示例]

你：在这个例子中，<T> 是什么意思？
```

### 3. 对比分析

对比相似概念或技术：

```
请问 TypeScript 的 interface 和 type 有什么区别？在什么情况下应该使用哪一个？
```

## 问答最佳实践

### 1. 提供代码片段

如果问题涉及代码，提供相关代码片段：

```typescript
// 我的组件代码
const UserProfile = ({ userId }: { userId: string }) => {
  const [user, setUser] = useState<User | null>(null);

  useEffect(() => {
    fetchUser(userId).then(setUser);
  }, [userId]);

  return <div>{user?.name}</div>;
};
```

### 2. 说明期望行为

描述你期望的结果：

```
我期望当 userId 变化时，用户数据会重新加载。但实际没有发生，是什么原因？
```

### 3. 提供错误信息

遇到错误时，提供完整的错误堆栈：

```
这是完整的错误信息：

TypeError: Cannot destructure property 'data' of 'undefined' as it is undefined.
    at processTicksAndRejections (node:internal/process/task_queues:96:5)
    ...
```

### 4. 说明已尝试的方法

说明你已经尝试过的解决方案：

```
我遇到了这个问题，已经尝试过：
1. 重新安装依赖
2. 清除缓存
3. 检查配置文件

但问题仍然存在，还有什么其他建议吗？
```

## 常见问题

### Q1: 为什么 OpenCode 的回答不准确？

**A:** 提高回答准确性的方法：

- **提供更多上下文**：描述问题的详细背景
- **引用相关代码**：使用 `@` 符号引用代码文件
- **明确期望**：清楚说明你想要什么样的答案
- **分步提问**：将复杂问题分解为多个简单问题

### Q2: 如何让 OpenCode 记住之前的对话？

**A:** OpenCode 会自动记住对话历史。如果发现它忘记了：

- 简单回顾之前的关键信息
- 重新提供必要的上下文
- 使用更明确的提问方式

### Q3: 可以同时问多个问题吗？

**A:** 建议一次只问一个问题，原因：

- **更专注**：每个问题都能得到充分的关注
- **更清晰**：避免问题之间的混淆
- **更准确**：减少上下文的复杂性

如果需要问多个相关问题，可以：

```
我有几个相关的问题：
1. 第一个问题...
2. 第二个问题...
3. 第三个问题...

请逐一回答这些问题。
```

### Q4: 如何判断 OpenCode 的回答是否正确？

**A:** 验证回答准确性的方法：

- **测试代码**：运行提供的代码示例
- **查阅文档**：对比官方文档
- **实践验证**：在实际项目中验证
- **多方求证**：使用其他资源交叉验证
